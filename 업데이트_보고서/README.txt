================================================================================
[명명 규칙]
================================================================================

[순번]_[종류]_v[버전].txt

예시:1.0.0.txt
  02_버그수정_v1.0.1.txt
  03_버그수정_v1.0.2.txt
  04_기능추가_v1.1.0.2.txt
  04_기능추가_v2.4.txt

순번: 시간 순서대로 01, 02, 03...
종류: 개선사항, 버그수정, 기능추가, 성능최적화 등
버전: semantic versioning (major.minor.patch)


================================================================================
[버전 관리]
================================================================================

Semantic Versioning:
1.0.0       → 초기 기능 개선 완료
v1.0.1       → 첫번째 버그 수정 (패치) ✓ 현재
v1.0.2       → 두번째 버그 수정 (패치)
v1.1.0       → 새로운 기능 추가 (마이너 버전)
v2.0.0       → 새로운 기능 추가 (마이너 버전)
v3.0         → 대규모 변경 (메이저 버전)

[사용 방법]
================================================================================

1) 새로운 보고서 추가 시:
   - 보고서_작성_가이드.txt 참고
   - 템플릿 사용하여 작성
   - 파일명 규칙 준수 ([순번]_[종류]_v[버전].txt)

2) 보고서 읽기:
   - 파일명의 버전과 종류로 내용 파악 가능
   - 최신 버전부터 순차적으로 읽을 것 추천

3) 버전 추적:
   - 모든 파일의 버전은 semantic versioning 따름
   - 패치 버전 = 버그 수정
   - 마이너 버전 = 새 기능/개선
   - 메이저 버전 = 큰 변화/리팩토링
